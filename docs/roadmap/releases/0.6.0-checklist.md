# Release 0.6.0 Checklist

## Purpose

This checklist is the execution and review baseline for `0.6.0`.

`0.6.0` is the milestone where Code Orb should turn `0.5.x` stabilization paths into explicit runtime contracts without expanding the foreground CLI into a broader product surface.

## Release Focus

The first priority for `0.6.0` is:

- make provider compatibility behavior explicit instead of tactical
- replace heuristic assistant-produced file writes with an auditable execution contract
- separate built-in tool registration from built-in tool execution
- define the minimum query-loop and tool-runtime contracts that `0.7.0` will extend next

For implementation order, use the priority plan below.

## Scope Discipline

- [x] `0.6.0` does not redesign the interactive CLI command surface introduced in `0.5.0`
- [x] `0.6.0` does not implement the full generic multi-iteration query loop reserved for `0.7.0`
- [x] `0.6.0` does not introduce a plugin marketplace or broad user-facing extension ecosystem
- [x] `0.6.0` does not require model-native tool calling as a release dependency
- [x] `0.6.0` remains focused on explicit runtime contracts and extensibility debt paydown

## Exit Rule

`0.6.0` is complete only when Code Orb can keep the current CLI usable while making provider compatibility, generated-edit execution, tool registration, and loop/runtime foundations explicit enough to support the next milestone without another hidden architectural reset.
