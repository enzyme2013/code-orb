# Release 0.7.0 Checklist

## Purpose

This checklist is the execution and review baseline for `0.7.0`.

`0.7.0` is the milestone where Code Orb should turn the `0.6.0` runtime contracts into a reusable turn-level query loop, a more explicit tool orchestration runtime, and a provider adapter baseline that owns transport and continuation behavior deliberately.

## Release Focus

The first priority for `0.7.0` is:

- replace the ordered task-specific branches in `BasicAgentEngine` with an explicit turn loop
- make the boundary between session-owned state and single-turn loop execution explicit enough that later shells and richer runtimes do not depend on one oversized engine object
- make tool, verification, retry, and stop outcomes re-enter runtime state through one shared path
- evolve the registry-backed tool executor into a loop-aware orchestration boundary
- establish a provider adapter baseline that owns transport, continuation, and fallback behavior explicitly

For implementation order, use the priority plan below.
