# Release 0.8.0 Checklist

## Purpose

This checklist is the execution and review baseline for `0.8.0`.

`0.8.0` is the milestone where Code Orb should turn the explicit `0.7.0` runtime skeleton into a CLI that is dependable enough for real local-repository work, without drifting into the broader terminal UX work owned by `0.9.0`.

## Release Focus

The first priority for `0.8.0` is:

- make repository guidance such as `AGENTS.md` part of normal runtime execution instead of a manual convention
- harden common inspect, plan, edit, verify, and report flows on top of the existing turn loop
- improve failure, approval, and mutation reporting so the CLI is understandable during real use
- prove usability through broader fixture, benchmark, and manual execution coverage rather than treating one demo path as sufficient

For implementation order, use the priority plan below.

## Kickoff Notes

At kickoff on 2026-04-09:

- the `0.8.0` release doc existed, but the checklist required by `docs/engineering/release-workflow.md` did not
- the current codebase has no explicit repository-instruction discovery or injection path for `AGENTS.md`
- existing regression gates are still concentrated around the current CLI e2e set and the `interactive-multi-turn` and `failing-test-fix` benchmarks
