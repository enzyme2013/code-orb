# Release 0.9.0 Checklist

## Purpose

This checklist is the execution and review baseline for `0.9.0`.

`0.9.0` is the milestone where Code Orb should turn the now-usable CLI into an interactive terminal experience that is materially easier to drive and inspect, without pulling core runtime semantics back into shell code.

## Release Focus

The first priority for `0.9.0` is:

- lock the shell/core ownership model for richer terminal UX before implementation drifts into runtime changes
- improve `orb chat` rendering, status surfaces, and session inspection
- make approval and diff review more legible in terminal flow
- prove the UX improvements through scripted interactive coverage and manual TTY validation

For implementation order, use the priority plan below.

## Kickoff Notes

At kickoff on 2026-04-09:

- the `0.9.0` release doc existed, but the checklist required by `docs/engineering/release-workflow.md` did not
- the current shell output is functional but still mostly sequential text that becomes hard to scan during longer interactive sessions
- current interactive inspection is limited to `/status` plus post-hoc `orb sessions list` and `orb sessions show`, with no richer shell-owned review path yet
- the existing architecture already assigns interaction, approval UX, and rendering to the shell layer, so `0.9.0` should preserve that boundary instead of reopening runtime semantics
